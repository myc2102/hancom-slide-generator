통합 디자인 시스템
원페이지 가이드라인

본 문서는 기획총괄의 관점에서 데이터/AI 플랫폼 및 관련 산출물의 일관성 있는 브랜드 아이덴티티를 유지하고, 효율적인 제작을 지원하기 위해 작성되었습니다. 모든 디자인 의사결정의 기준점(Single Source of Truth)으로 활용됩니다.

01
Foundation — 기초 체계
01-01

개요 및 목적

전문성과 신뢰성을 시각적으로 전달하고, 디자이너·개발자 간의 커뮤니케이션 비용을 최소화하는 것을 목표로 합니다. 기획서의 의도가 시각 결과물로 왜곡 없이 전달되도록 돕습니다.

디자인 철학
데이터 기반의 논리적 레이아웃, 미니멀리즘, 고대비 다크 비주얼
적용 범위
브로셔, 제안서, 프레젠테이션, 웹 대시보드, UI/UX 화면 설계
01-02

디자인 원칙

원칙 1 — 콘텐츠 불변의 법칙

절대 규칙

디자인 작업 과정에서 원본 기획 콘텐츠의 내용, 순서, 논리적 흐름을 임의로 줄이거나 변형하는 것을 엄격히 금지합니다. 플레이스홀더 수량과 데이터 항목은 1:1로 매칭되어야 합니다.

원칙 2 — 시각적 위계 (Visual Hierarchy)

가장 중요한 정보가 눈에 띄도록 크기, 색상, 여백을 조절합니다.

레벨용도추천 스타일
Level 1핵심 타이틀32px+ / Bold 800 / 그라데이션
Level 2서브 섹션20px+ / Bold 700 / Primary Color
Level 3본문 / 상세15px / Regular 400 / Secondary Color

원칙 3 — 종교적·문화적 중립성

데이터 파이프라인이나 노드를 표현할 때 특정 종교나 사상을 연상시키는 형태(십자가, 특정 별 모양 등)를 피하고 기하학적인 중립 형태를 사용합니다.

01-05

여백과 레이아웃

개발 효율을 높이기 위해 4px / 8px 베이스 그리드 시스템을 엄격하게 적용합니다.

  • 요소 간 간격: 8px, 16px
  • 컨테이너 패딩: 24px, 32px
  • 섹션 여백: 64px, 80px
02
Visual Identity — 시각 정체성
02-01

브랜드 톤앤매너

혁신적이고 미래지향적인 기술 기업의 이미지를 전달합니다.

DO — 지향
  • 딥 다크 배경 + 선명한 액센트 컬러
  • 여백을 충분히 활용한 미니멀 레이아웃
  • 기하학적이고 정돈된 선형 그래픽
DON'T — 회피
  • 불필요한 3D 장식이나 복잡한 일러스트
  • 광원이 불일치하는 무분별한 합성
  • 형광색의 과도한 남용
02-02

다크 테마 시스템

배경의 명도(Brightness) 차이만으로 정보의 깊이감(Elevation)을 표현합니다. 순수 블랙(#000000)은 피하고 약간의 푸른빛이 도는 틴트 다크(#0a0a0f)를 사용합니다.

02-03

그라데이션 & 이펙트

빛 번짐(Glow) 효과를 통해 사이버네틱한 느낌을 주지만, 텍스트 가독성을 해치지 않는 선에서 사용합니다.

/* 텍스트 그라데이션 */ background: linear-gradient(135deg, #e8e8ed, #4a9eff, #00dbc8); -webkit-background-clip: text;
02-04

아이콘 & 오브젝트

플랫(Flat)한 라인 아이콘을 기본으로 하며, 핵심 기술 구조도나 아키텍처를 표현할 때는 점과 선(Node & Edge) 기반의 추상적 네트워크 그래픽을 활용합니다.

02-05

모션 & 인터랙션

대시보드 환경에서 데이터의 실시간 동기화를 표현하기 위한 애니메이션입니다.

실시간 데이터 수신 중...
03
Component Library — 컴포넌트
03-01

카드 & 컨테이너

정보를 그룹화하는 기본 요소입니다. 보더(Border)를 1px로 얇게 유지하여 샤프한 느낌을 줍니다.

03-02

테이블 & 데이터

대량의 데이터를 표시할 때 테이블 헤더는 `Uppercase`와 넓은 자간(Tracking)을 적용하여 구분감을 확실히 줍니다.

03-03

콜아웃 & 알림

Info

일반적인 안내 문구입니다.

Warning

주의가 필요한 설정이나 제약사항입니다.

03-04

태그 & 뱃지

Processing AI Model Completed Error
03-05

코드 블록 스타일

API 연동 가이드나 개발 문서를 표기할 때 사용하는 다크 테마 코드 스니펫 양식입니다.

03-06

대시보드 & 맵 UI

지리정보(GIS)나 노드 맵을 표현할 경우, 기본 베이스를 다크 벡터 스타일로 통일하여 그 위에 올라가는 분석 데이터(Bounding Box 등)의 식별력을 높입니다.

  • 일반 객체 탐지: Cyan (#00dbc8) Solid Line
  • 위험 객체 탐지: Rose (#f43f5e) Pulse Line
04
Production — 제작 가이드
04-01

브로셔/문서 가이드

HTML 기반으로 문서를 퍼블리싱한 뒤 PDF로 변환하는 방식을 권장합니다. A4 세로 비율에 맞춘 컨테이너를 사용합니다.

04-02

프레젠테이션 가이드

1920 × 1080 (또는 데이터 밀도를 위한 1920 × 850) 해상도를 표준으로 합니다.

04-03

포스터 & 비주얼

정사각형(2000x2000) 비율을 기본으로 하며, 추상적인 데이터 흐름을 시각화합니다.

04-04

데이터 시각화 HTML

엑셀로 정리된 데이터를 HTML 테이블로 퍼블리싱할 때, 최대 높이 제한(800px)을 두어 뷰포트 오버플로우를 방지합니다.

05
Process — 프로세스
05-01

AI 이미지 개선 가이드

AI로 생성한 소스에서 과도한 노이즈, 인위적인 글로우 효과, 엉뚱한 연결선 등을 포토샵이나 코드 필터로 제거하는 후처리 단계를 반드시 거칩니다.

05-02

Canvas 아트 기법

HTML5 Canvas API를 이용하여 추상적인 백그라운드나 파티클 효과를 렌더링한 후 PNG로 추출하는 기법을 적극 활용합니다.

05-03

자동화 파이프라인

  1. 마크다운(MD) 또는 엑셀 기획서 작성
  2. AI(Gemini)를 통한 JSON 파싱 및 구조화
  3. 스크립트(python-pptx 등)를 통한 슬라이드 자동 생성
05-04

GitHub 운영 가이드

이 디자인 가이드는 GitHub Pages를 통해 팀 전체에 실시간 공유됩니다. 수정이 필요할 경우 리포지토리의 `index.html`을 수정 후 Commit 하세요.

05-05

최종 체크리스트

  • 원본 기획서의 데이터가 누락 없이 100% 반영되었는가?
  • 정의된 컬러 CSS 변수(`--accent-blue` 등)를 올바르게 사용했는가?
  • 모바일 뷰포트 대응 및 브라우저 호환성을 확인했는가?
