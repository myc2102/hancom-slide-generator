디자인 원칙

원칙 1 — 콘텐츠 불변의 법칙

절대 규칙

원본 콘텐츠의 내용, 순서, 표현을 절대로 줄이거나 변형하지 않습니다. 브로셔 제작 시 확인된 핵심 원칙으로, 이미지 플레이스홀더 수량까지 정확히 맞춰야 합니다.

이 원칙은 AI 컨퍼런스 브로셔 제작에서 여러 번의 검수를 통해 확립되었습니다. 제품 스크린샷 슬롯이 줄어든 것, 이미지 슬롯이 누락된 것, 섹션 라벨이 빠진 것 등 모든 케이스가 검수에서 발견되었고, 이를 원칙으로 고정했습니다.

원칙 3 — 대비(Contrast)의 극대화

포스터 작업에서 "배경과 오브젝트의 대비차이를 더 내달라"는 피드백에서 확립된 원칙입니다. 다크 배경에서는 오브젝트의 글로우(glow), 채도(saturation), 밝기(brightness)를 적극적으로 올려 시각적 충격을 줍니다.

  • 배경: 가능한 깊고 어둡게 (#0a0a0f ~ #111118 수준)
  • 오브젝트: 밝고 선명한 색상 + 글로우 이펙트로 강조
  • 비네트: 가장자리를 더 어둡게 하여 중심부에 시선 집중

원칙 4 — 문화적 민감성

주의 사항

특정 종교·문화를 연상시키는 형태를 사용하지 않습니다. 비주얼 작업에서 중앙의 오브젝트가 특정 심볼로 보인다는 피드백을 받아 형태를 변경한 사례가 있습니다.

원칙 5 — AI 생성 느낌 배제

AI가 생성한 이미지의 전형적 문제점을 인지하고 회피합니다.

  • 너무 많은 요소를 한 화면에 억지로 합성하지 않기
  • 광원 방향을 일관되게 유지하기
  • 빛나는 노드/연결선 — AI 느낌의 주범, 최소화
  • 균일한 질감 — 필름 그레인, 노이즈 등으로 자연스러움 추가

브랜드 톤앤매너

HANCOM AI·AX 비주얼 아이덴티티

AI 전환(AX) 사업과 한컴 생산성 플랫폼을 아우르는 비주얼 언어입니다.

슬로건

AI Transformation
Connecting AI, Workflow, and Business

이 슬로건은 모든 비주얼 작업의 핵심 컨셉입니다. AI 모델 → 워크플로우 자동화 → 비즈니스 성과를 연결하는 AI 전환(AX)을 시각적으로 표현합니다.

톤앤매너 Do / Don't

DO — 지향
  • 딥 다크 배경 + 선명한 액센트 컬러
  • 추상적이고 드라마틱한 데이터 비주얼
  • 깔끔하고 미니멀한 Apple 스타일 레이아웃
  • 명확한 시각적 위계
  • 영역별 컬러 코딩 (블루/시안/앰버)
  • 필름 그레인, 비네트로 깊이감 추가
DON'T — 회피
  • AI 생성 특유의 빛나는 노드/연결선
  • 한 화면에 너무 많은 요소 강제 합성
  • 광원 방향이 제각각인 이미지
  • 종교적·문화적 민감 형태 연상
  • 보라색 그라디언트 + 흰 배경 (AI 클리셰)
  • Inter, Roboto 등 제네릭 폰트 사용

다크 테마 시스템

모든 산출물은 기본적으로 다크 테마를 사용합니다. 인쇄물의 경우 라이트 테마로 변환할 수 있지만, 디지털 산출물(HTML, 프레젠테이션, 포스터)은 다크 테마가 기본입니다.

배경 레이어 구조

L0 Base
L1 Surface
L2 Card
L3 Elevated
L4 Border

각 레이어는 약 6–10의 밝기 차이를 두어 깊이감을 형성합니다. 배경→카드→보더 순으로 점진적으로 밝아집니다.

다크 테마에서의 텍스트 가독성

  • Primary (#e8e8ed): 타이틀, 강조 텍스트 — 배경 대비 최소 7:1
  • Secondary (#9898a8): 본문 텍스트 — 배경 대비 최소 4.5:1
  • Muted (#666678): 보조 정보, 캡션 — 장식적 용도에만 사용

글로우 이펙트

여러 개의 radialGradient 타원을 겹쳐서 글로우 효과를 생성합니다. 각 타원은 서로 다른 색상/투명도를 가지며, globalAlpha로 밀도를 조절합니다.

// 필름 그레인 — 모든 Canvas 포스터에 마지막 단계로 적용 const imgData = ctx.getImageData(0, 0, W, H); const d = imgData.data; for (let i = 0; i < d.length; i += 4) { const n = (Math.random() - 0.5) * 10; d[i] = Math.max(0, Math.min(255, d[i] + n)); d[i+1] = Math.max(0, Math.min(255, d[i+1] + n)); d[i+2] = Math.max(0, Math.min(255, d[i+2] + n)); } ctx.putImageData(imgData, 0, 0);

05-02

Canvas 아트 기법

HTML Canvas 2D로 포스터/비주얼을 제작할 때 사용하는 기법 목록입니다.

렌더링 순서 (레이어 스택)

  1. 배경 그라데이션 — createRadialGradient로 깊은 다크 배경
  2. 파티클 필드 — 600~1000개의 작은 원, 랜덤 크기/투명도
  3. 밝은 포인트 + 글로우 — 15~20개, 십자형 라인 + 밝은 원
  4. 배경 클라우드 — 3~5개의 타원형 radialGradient, 각각 다른 색상
  5. 핵심 오브젝트 — AI 브레인, 워크플로우, 대시보드 등 (각각 독립 함수)
  6. 오브젝트 간 연결선 — 점선 또는 얇은 라인으로 관계 표현
  7. 영역 구분선 — 수평 라인, 양쪽 끝 transparent 페이드
  8. 부유 파티클 — 영역별 컬러(블루/시안/앰버), 200~300개
  9. 비네트 — 가장자리 어둡게 하는 radialGradient
  10. 필름 그레인 — getImageData + 랜덤 노이즈 (마지막 단계)

핵심 함수 패턴

글로우 클라우드 함수

function glowCloud(cx, cy, rx, ry, c1, c2, alpha) { ctx.save(); ctx.globalAlpha = alpha; const g = ctx.createRadialGradient(cx, cy, 0, cx, cy, Math.max(rx, ry)); g.addColorStop(0, c1); g.addColorStop(0.5, c2); g.addColorStop(1, 'transparent'); ctx.fillStyle = g; ctx.beginPath(); ctx.ellipse(cx, cy, rx, ry, 0, 0, Math.PI * 2); ctx.fill(); ctx.restore(); }

비네트 함수

const vig = ctx.createRadialGradient(W/2, H/2, W*0.2, W/2, H/2, W*0.85); vig.addColorStop(0, 'transparent'); vig.addColorStop(0.6, 'rgba(0,0,5,0.2)'); vig.addColorStop(1, 'rgba(0,0,5,0.7)'); ctx.fillStyle = vig; ctx.fillRect(0, 0, W, H);
